JS Keywords

JS keywords are used to identify actions to be performed.

JS Identifiers

Best practices for naming identifiers are:

  • The first character must be a letter, _, or $.
  • Numbers are not allowed as the first character.
  • Identifiers are case sensitive.
  • Use lower camel case.
    • ex: lowerCamelCase
  • Hyphens are not allowed.

Character Set

JS uses the Unicode character set.

JS Comments

Use // for single line comments or /* */ for multi-line comments.

JS Variables

Identifiers (names) must begin with:
  • A letter, A-Z or a-z
  • $ is used as an alias for the main function in a JS library.
    • $ is the main function used to select HTML elements.
    • $("p"); means select all p elements.
    • JQuery is a JS library.
  • Underscore, _
    • _ is used as an alias for private (hidden) variables.

These are treated like letters.

Arrays

  • Arrays contain multiple items.
  • Array indexes are zero-based.
  • Arrays are written with square brackets.

Objects

  • Objects are written with curly brackets { }.
  • Its properties are written as name:value pairs.
  • ex: var x = {first:"Jane", age:25, hairColour:"hazel"};
  • Object x has 3 properties: first, age, and hairColour.

Empty Values

Null

Null's data type is Object. You can empty an object by setting its value to null. It's type is still an object. Null and Undefined have equal value but have different types.

    There are 3 methods for extracting part of a string:

    • slice(start, end)
    • substring(start, end)
    • substr(start, length)

    slice() Method

    Returns the extracted part in a new string.
    2 parameters: start position, end position (end not included)

    substring() Method

    Substring cannot accept negative indexes. If you omit the second parameter, the method will slice out the rest of the string like slice().

    substr() Method

    The second parameter specifies the length of the extracted part.

    Omitting the second parameter slices out the rest of the string.
